# Рекомендательная система сервисов для научных рабочих процессов с использованием графового трансформера и учёта пользовательских предпочтений (Бенчмарк Galaxy)

## Введение
Разработка научных рабочих процессов (workflows) для обработки больших данных, в частности биоинформатических данных, требует выбора подходящих инструментов из тысяч доступных. Даже опытные исследователи тратят много времени на поиск совместимых сервисов, а новички часто делают ошибочный выбор. Существующие системы рекомендаций (например, Galaxy Tool Recommender или BTR) в основном опираются на линейные последовательности инструментов или простые графы, не учитывая ни полную структуру DAG, ни предпочтения пользователей из одной предметной области.

В данной работе представлена **Preference-Aware Graph Transformer (PA-Graph Transformer)** – новая модель для рекомендации следующего сервиса в рабочем процессе, которая сочетает:
1. **Графовое представление** рабочего процесса как направленного ациклического графа (DAG), что позволяет учитывать разветвления и слияния потоков данных.
2. **Graph Transformer** – архитектуру, использующую relation-aware self-attention с учётом расстояний между узлами в глобальном графе всех инструментов.
3. **Учёт предпочтений пользователей (user preference aggregation)** – для каждого пользователя (автора рабочего процесса) строится вектор частот использования сервисов, затем находятся похожие пользователи (kNN по косинусному сходству) и их история выбора следующего шага агрегируется в дополнительное распределение вероятностей. Финальный прогноз комбинирует выход Graph Transformer и этот коллаборативный сигнал.

## Сравнение с существующими подходами
Мы провели сравнение PA-Graph Transformer с базовыми моделями на наборе данных Galaxy.

### Важное уточнение по поводу моделей-бенчмарков:
> [!IMPORTANT]
> Для существующих опубликованных моделей (таких как BTR и GTR) общедоступный исходный код авторов представлен **исключительно для линейного представления данных** (linear sequences/paths). Тем не менее, в оригинальной статье BTR [Green et al., 2023] приводится теоретическое и экспериментальное обоснование превосходства графового подхода над линейным. В связи с этим в данном проекте мы реализовали как классические линейные версии (BTR sequence/path), так и адаптировали подходы для работы с восстановленными графами.

В качестве бенчмарков использованы:
* **GTR (Galaxy Tool Recommender, Kumar et al.)** – модель GRU4Rec на линейных последовательностях (Galaxy-style).
* **BTR (Bioinformatics Tool Recommendation, Green et al.)** – SR-GNN на линейных последовательностях (BTR sequence/path), а также вариант SR-GNN на восстановленных графах.
* **GPT** – декодерный трансформер (GPTRecommender), обученный на последовательностях.
* **SASRec, GRU4Rec** – классические модели session-based рекомендаций, адаптированные под наши DAG-контексты.
* **Graph Transformer** (без учёта пользователей) на восстановленных графах.
* **User-kNN + Graph Transformer** – коллаборативная фильтрация поверх базового графового трансформера (с фиксированным смешиванием).
* **PA-Graph Transformer** – наша модификация с динамическим взвешиванием на основе силы сходства предпочтений пользователя.

## Ключевые выводы
1. **Графовое представление (восстановленные DAG)** значительно превосходит любые линейные последовательности (Galaxy-стиль или BTR-стиль) при прочих равных условиях. Это обусловлено тем, что DAG сохраняет параллельные ветки и слияния данных, которые теряются в линейной развёртке.
2. **Graph Transformer** сам по себе даёт высокие результаты благодаря механизму relation-aware внимания, но добавление **user preference aggregation (PA-Graph Transformer)** стабильно повышает точность, полноту и F1-меру, доказывая ценность учёта индивидуального контекста пользователя.
3. Простая коллаборативная фильтрация (**User-kNN + Graph Transformer**) также улучшает базовый графовый трансформер, что подтверждает важность персонализации, однако динамическое шлюзование в PA-Graph Transformer работает точнее.
4. Модель **BTR (SR-GNN на последовательностях)** и **GTR (GRU4Rec на последовательностях)** существенно уступают графовым подходам, так как не способны моделировать сложные топологические структуры.
5. Современные LLM (**GPT**) без глубокого и специализированного файнтюнинга не могут на равных конкурировать с узкоспециализированной графовой архитектурой на данной задаче.

## Структура архива
Внутри архива находятся следующие файлы, необходимые для запуска:
* `recom2_tsv.ipynb` — основная рабочая тетрадка Jupyter Notebook, содержащая полный цикл подготовки данных, обучения всех моделей и визуализации результатов на датасете Galaxy.
* `directed_dag_models.py` — библиотека, содержащая базовые реализации моделей (SR-GNN, GRU4Rec), процедуры их обучения и функции подсчёта метрик.
* `modern_sequence_models.py` — библиотека, реализующая архитектуры SASRec, GPT и Graph Transformer.
* `cycle_handling.py` — алгоритмы разрешения циклических зависимостей при сборке DAG из линейных логов (стратегия `remove_weakest`).
* `tsv_workflow_benchmark_utils.py` — утилиты для парсинга EuGalaxy TSV файла, восстановления DAG, извлечения путей и подготовки тензоров для обучения.
* `final_comparison.png` — финальное изображение общего сравнения моделей.
* `README.md` — данное руководство.

## Автоматическая загрузка данных
> [!NOTE]
> Исходный датасет `worflow-connection-20-04.tsv` имеет размер около 25 МБ. При первом запуске ячеек в `recom2_tsv.ipynb` тетрадка **автоматически скачает** его из официального репозитория авторов в созданную папку `data/`. Если вы хотите работать в офлайн-режиме, просто создайте директорию `data/` в корне проекта и поместите туда файл `worflow-connection-20-04.tsv`.

## Инструкция по запуску
1. Убедитесь, что установлены все необходимые зависимости:
   ```bash
   pip install torch torch-geometric networkx numpy pandas scikit-learn matplotlib seaborn requests
   ```
2. Запустите Jupyter Notebook или Jupyter Lab:
   ```bash
   jupyter notebook
   ```
3. Откройте тетрадку `recom2_tsv.ipynb` и выполните все ячейки по порядку (Cell -> Run All). Обучение моделей на полном датасете может занять около 1-2 часов на современной видеокарте (например, RTX 3060). Если вы хотите быстро проверить работоспособность, установите в конфигурационной ячейке `FAST_PRESET = True` (это уменьшит количество эпох и ограничит размер выборки).

## Заключение
Предложенная модель **Preference-Aware Graph Transformer** успешно решает задачу интеллектуальной подсказки сервисов на крупномасштабном биоинформатическом датасете Galaxy, превосходя существующие линейные и графовые аналоги. Это открывает перспективы внедрения системы в качестве ассистента исследователя на публичных серверах Galaxy, повышая эффективность проектирования сложных научных конвейеров.
